What is a Pentesting lab ?


As you know any intrusions or fraudulent retention in a STAD (système de traitement automatisé de données) is criminally reprehensible, so we can't test our skills in real conditions (victim/attacker). The only way to test our skill is to create a device vulnerable to our exploits
This device will be a Pentesting Lab, a virtual or physical environnement that is vulnerable to some exploit, it permit to ethical hacker to train their skills, without being in the illegality


The image of a rpi that can be used as a server


Your Pentesting lab could be materialised by a vulnerable server or a raspberry pi for exemple. Everything depends on your needs ...

Physical Pentesting Lab


A Physical Pentesting Lab is a lab that could take the aspect of a server (hosted on the local network on a raspberry pi for exemple), or just a vulnerable device.


What's a vulnerability ? :

In the domain of computer security, a vulnerability is a weakness in an informatic system that allows an attacker to compromise the integrity of that system. To "exploit" these vulnerability we have some programms, named "exploits" that allow a personn or a malicious software to exploit a computer security breach in a informatic system.


+ Positive Aspects:

- Allow you to see the direct impact on the victim screen and the comportement of your exploit on a real machine.

- You can interact with the machine physicaly, therefore the exploitation is easier


- Negative Aspects:

- You can't control the performances of your machine that is limited by his hardware configuration






Virtual Pentesting Lab


A Virtual Pentesting Lab is a lab that is virtualised or that is in a cloud, into a physical machine, and that use the ressources allocated by this machine. Some pentesting labs can also be virtualized as sites


What's the difference between a cloud and a virtual machine ? :

Virtualization is a technology that separates hardware functions of a machine, to create new machines (virtual machines) by allocating them a part of this hardware functions, while clouds rely on this division. So the existence of a cloud depend on the existence of a virtual machine.


+ Positive Aspects:

- if you destroy in his integrality your virtual machine, you'll be able to restore a backup

- you can create your pentesting lab on your principal computer by allocating to it a part of your pc's performances (without taking all the ressources)

- Some sites also offer to perform XSS cross scripting injection and web based pentesting. This type of virtual labs allow you to perform a huge variety of exploits


- Negative Aspects:

- When you have to setup your virtual machine, it's sometimes hard to find the right files, not all the OS files can be used as virtual machine os or raspberry pi os, we need a specific type of files
